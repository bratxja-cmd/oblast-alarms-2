# Дашборд повітряних тривог по областях

Збирає тривоги по 27 областях (усі області + Київ + Крим + Севастополь) з alerts.in.ua, рахує тривалість і кількість,
показує рейтинг + деталізацію. Дані за останній місяць, далі накопичуються щодня.

## Файли

```
config.py          # список областей (uid + назва)
collect.py         # збір з API -> data/alerts.db
build_stats.py     # база -> data/stats.json (зі злиттям перекриттів)
index.html         # дашборд
.github/workflows/update.yml  # щоденне автооновлення + публікація
data/              # тут зʼявляться alerts.db і stats.json
```

---

## Розгортання на GitHub (з нуля)

### 1. Створіть репозиторій
github.com → New repository → назва (напр. `oblast-alarms`) → **Public** →
без README → Create.

### 2. Завантажте файли
Add file → Upload files → перетягніть **увесь вміст** цієї теки.
Важливо: тека `.github` (з крапкою) має теж потрапити. Якщо перетягуванням не
виходить — створіть файл вручну: Add file → Create new file → у назві впишіть
`.github/workflows/update.yml` → вставте вміст із вашого файлу → Commit.
Commit changes.

### 3. Отримайте API-токен
https://alerts.in.ua/api-request → заявка → токен.

### 4. Додайте токен у секрети
Settings → Secrets and variables → Actions → вкладка **Secrets** →
New repository secret → Name рівно `ALERTS_TOKEN`, Secret = ваш токен → Add.

### 5. Дозволи
Settings → Actions → General → Workflow permissions →
**Read and write permissions** → Save.

### 6. Увімкніть Pages
Settings → Pages → Source → **GitHub Actions**.

### 7. Перший запуск
Actions → workflow «Оновлення тривог по областях» → Run workflow.
Дочекайтесь, поки обидва кроки (collect і deploy) стануть зеленими (~14 хв — 27 областей із паузами).

### 8. Готово
Посилання зʼявиться у Settings → Pages:
`https://ВАШ-НІК.github.io/oblast-alarms/`

## Як оновлюється

Щодня о 05:00 UTC (08:00 за Києвом) workflow сам збирає свіжі дані й перевидає
сайт. Запуск інколи із затримкою до пів години — це нормально для GitHub.
Оновити негайно: Actions → Run workflow.

База `alerts.db` накопичується: щодня докладаються нові тривоги, дублі
відсікаються за змістом (область + час початку). Історія росте вперед.

## Налаштування

- **Змінити список областей** — редагуйте `config.py` (uid з довідника alerts.in.ua, області мають uid 3–31).
- **Змінити час оновлення** — рядок `cron` у `.github/workflows/update.yml`
  (формат: хвилина година * * *, час за UTC).

## Обмеження (важливо)

- Рівень **областей**, не громад: історичний API alerts.in.ua по громадах/районах
  повертає 404, історія доступна лише по областях.
- Дані — за **останній місяць** від моменту збору, далі накопичуються. Глибшу
  історію заднім числом цей API не віддає.
- Числа можуть відрізнятися від інших сервісів через різну методологію
  підрахунку перекриттів — це нормально.

## Якщо щось не так

- Сайт порожній / «дані ще не зібрані» — перевірте, що останній запуск в Actions
  зелений; відкрийте `https://ВАШ-НІК.github.io/oblast-alarms/data/stats.json` —
  чи є там дані.
- collect падає — розгорніть крок у Actions, гляньте помилку (найчастіше токен).
- НЕ редагуйте `index.html` вручну по дрібницях — там JS, одна помилка ламає все.
