# Дашборд повітряних тривог по областях України

Візуалізація статистики повітряних тривог по областях, районах, громадах і містах України на основі даних [alerts.in.ua](https://alerts.in.ua).

🔗 **Сайт:** https://bratxja-cmd.github.io/oblast-alarms-2/

## Що показує дашборд

- Рейтинг областей за сумарною тривалістю або кількістю тривог
- Розбивку по місяцях, з можливістю переглянути окремий місяць
- Деталізацію на рівні районів, громад і міст
- Розподіл за типами тривог (повітряна тривога, артобстріл, вуличні бої, хімічна/ядерна загроза)

## Джерело даних

Дані отримуються через офіційне історичне API [alerts.in.ua](https://devs.alerts.in.ua/) (`/v1/regions/{uid}/alerts/{period}.json`). Збір відбувається щодня автоматично (GitHub Actions), дані накопичуються з часом — історичний ендпоінт API повертає лише останній місяць, тож повна історія будується поступово, зберігаючи попередні збори.

## Методологія підрахунку

Дані про тривоги мають кілька особливостей, які варто розуміти при читанні графіків:

**Область ≠ сума районів.** Підсумок області — це об'єднані в часі події (одна тривога, що охопила кілька районів одночасно, рахується один раз). У розбивці по районах та сама тривога рахується в кожному районі окремо. Тому сума років по районах у деталях зазвичай **більша** за підсумок області вгорі — це очікувано, не помилка.

**Типи тривог показані валовим числом.** Розбивка за типами (повітряна тривога, артобстріл тощо) рахує кожне оголошення на кожному рівні (район, громада, місто) окремо, без об'єднання. Тому сума по типах може суттєво перевищувати загальну кількість тривог області.

**Області без активних даних приховані.** Якщо область не має жодної коректно зафіксованої тривоги (наприклад, через окупацію та відсутність стабільного джерела сигналу), вона не показується в рейтингу. Список таких областей — унизу сторінки. Це означає відсутність даних, а не відсутність тривог.

**Тривоги без зафіксованого завершення, що зависли на тижні чи роки**, автоматично відфільтровуються як технічні артефакти джерела даних (відсутність сигналу відбою), а не як реальні події.

## Технічний стек

- **Збір даних:** Python + SQLite (`collect.py`)
- **Обробка:** Python (`build_stats.py`) — злиття тривог, що перекриваються в часі, побудова дерева область → район/громада/місто
- **Відображення:** статичний HTML/CSS/JS (`index.html`), без фреймворків і збірки
- **Автоматизація:** GitHub Actions (щоденний запуск за розкладом)
- **Хостинг:** GitHub Pages

## Локальний запуск

```bash
git clone https://github.com/bratxja-cmd/oblast-alarms-2.git
cd oblast-alarms-2
python3 -m http.server 8000
```

Відкрити `http://localhost:8000` у браузері. Для оновлення даних потрібен токен API alerts.in.ua (`ALERTS_TOKEN`) — див. [документацію API](https://devs.alerts.in.ua/) для отримання доступу.

## Ліцензія та атрибуція

Дані належать [alerts.in.ua](https://alerts.in.ua) і використовуються відповідно до умов їхнього API. Код проєкту — вільний для форку й адаптації.
